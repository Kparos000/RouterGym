# Shared Drive Access Procedure

**Category:** Storage  

## Description
Granting and managing access to on-prem or network file shares.

## Typical Ticket Patterns
- “Need access to X:\\ drive”
- “Cannot map shared drive”
- “Access denied on shared folder”

## Resolution Steps
1. Confirm share path and owning team.
2. Validate manager/data owner approval.
3. Add user to appropriate AD or NAS group and update permissions.
4. Guide user to map the drive or confirm automatic mapping via login script.
5. Ask user to log off/on and test access.

## When to Escalate
- Sensitive shares (finance, HR, legal).
- Whole team suddenly loses share access.
- NAS or file server outage suspected.
